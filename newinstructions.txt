Alright — let’s do this like you’ve never deployed anything before. No assumptions, no gaps. By the end, your system will be **live on the internet**.

We’ll use the easiest professional setup:

* **Frontend (React)** → Vercel
* **Backend (Node.js)** → Render
* **Database** → MongoDB Atlas

---

# 🧭 STEP 0: What you need (once only)

Make sure you have:

* A Node.js installed
* A Git account
* A GitHub account

If not, install Node.js and create GitHub account first.

---

# 🧱 STEP 1: Prepare your project

Split your project into 2 folders:

```
event-booking/
 ├── frontend/
 └── backend/
```

---

## 🔹 Backend setup

Inside `backend/`:

### 1. Create server file

```bash
npm init -y
npm install express mongoose cors dotenv
```

Create:

```
backend/server.js
backend/.env
```

### 2. Add `.env`

```
PORT=5000
MONGO_URI=your_mongodb_connection
JWT_SECRET=your_secret_key
```

---

# 🌐 STEP 2: Create Database (MongoDB Atlas)

Go to:
👉 MongoDB Atlas

### Steps:

1. Create free account
2. Create **Cluster (free tier)**
3. Create database user (username/password)
4. Click **Connect → Drivers → copy connection string**

Example:

```
mongodb+srv://user:password@cluster.mongodb.net/eventdb
```

Paste into `.env`

---

# 📤 STEP 3: Upload code to GitHub

### 1. Initialize git

```bash
git init
git add .
git commit -m "first commit"
```

### 2. Push to GitHub

```bash
git remote add origin https://github.com/yourname/event-booking.git
git push -u origin main
```

---

# 🚀 STEP 4: Deploy Backend (Render)

Go to:
👉 Render

### Steps:

1. Click **New → Web Service**
2. Connect your GitHub repo
3. Select backend folder

### Settings:

* Build Command:

```
npm install
```

* Start Command:

```
node server.js
```

### Add Environment Variables:

* MONGO_URI
* JWT_SECRET

Click **Deploy**

👉 After deploy, you get:

```
https://your-app.onrender.com
```

---

# 🎨 STEP 5: Deploy Frontend (Vercel)

Go to:
👉 Vercel

### Steps:

1. Import GitHub repo
2. Select `frontend/` folder

### Important:

Update your frontend API calls:

```js
fetch("https://your-app.onrender.com/events")
```

(not localhost anymore!)

Click **Deploy**

👉 You get:

```
https://your-app.vercel.app
```

---

# 🔗 STEP 6: Connect Everything

Now your system is live:

* Frontend (Vercel) → talks to → Backend (Render)
* Backend → talks to → MongoDB Atlas

---

# ⚠️ STEP 7: Fix common beginner issues

### ❌ “CORS error”

Fix in backend:

```js
app.use(cors({ origin: "*" }));
```

---

### ❌ “Database not connecting”

Check:

* Username/password correct
* IP whitelist = `0.0.0.0/0` in MongoDB Atlas

---

### ❌ “App crashes on deploy”

Check Render logs:

* Missing env variables
* Wrong start command

---

# 🔐 STEP 8: Make it production-safe (IMPORTANT)

Before real users:

* Use strong JWT secret
* Never upload `.env` to GitHub
* Enable HTTPS (automatic on Vercel/Render)
